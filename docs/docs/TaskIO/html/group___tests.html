Main interface to the compiled executable.

This function serves as the main entry point to the compiled executable. It handles various test commands and executes corresponding tests.

Usage

Command Line Operation

Once tests.exe is compiled, you can get a list of the test options using the help command:

bin\tests.exe help

The different test command options are listed below:

  • readconfig - Read and print the configuration from config.yaml.
  • staticwrite - Checks that a static .txt file can be created (C:/Data/TaskLogs/test.txt).
  • readcursor <subject> <YYYY> <MM> <DD> <index> - Read and print the values from the specified cursor log.
  • readstate <subject> <YYYY> <MM> <DD> <index> - Read and print the values from the specified state log.
  • writecursor <subject> - Create and write a sample cursor log.
    • If C:/Data/TaskLogs (or other folder specified in bin/config.yaml) does not exist yet, the first time running may throw an error but you should see the folder is created. Re-running should then produce the intended behavior.
  • writestate <subject> - Create and write a sample state log.
    • If C:/Data/TaskLogs (or other folder specified in bin/config.yaml) does not exist yet, the first time running may throw an error but you should see the folder is created. Re-running should then produce the intended behavior.

Example Testing

Start with readconfig which will also show you where the logger wants to save/load files on your local system.

bin\tests.exe readconfig

Next, you can writecursor which will generate some random interpolated cursor paths that can be used for later tests.

bin\tests.exe writecursor MyTest

This should generate a file in the save folder indicated by the readconfig step. If there is an error, try re-running as the error is probably just due to the folder not existing to begin with. Next, try reading back what you just generated. Assuming the date is 2024-02-08, then to access the log file you just created, follow syntax below:

bin\tests.exe readcursor MyTest 2024 2 8 1

Run the writecursor test again.

bin\tests.exe writecursor MyTest

You will notice that it auto-increments the file index, if you check the log save location returned by bin\tests.exe readconfig.
To read back the new file, just increment the index accordingly:

bin\tests.exe readcursor MyTest 2024 2 8 2
